# Epplaa Architecture — v4.2 Amendment

- **Status**: Active
- **Date**: 2026-04-30
- **Supersedes**: portions of v4.1 listed in §"Deviations from v4.1"
- **Authoritative spec**: `attached_assets/Epplaa_Architecture_Sprint_Plan_v4.1_*.md`

This amendment records the deviations from the v4.1 architecture spec
required by the current code base and by the team's chosen technology
stance. It is *additive* to v4.1: where this amendment is silent, the
v4.1 spec is binding.

## 1. Deviations from v4.1

The complete rationale for each deviation lives in the cited ADR.

| v4.1 spec | v4.2 amendment | Rationale |
| :--- | :--- | :--- |
| Mobile = Flutter (Riverpod, go_router, dio, drift, flutter_webrtc, video_player + chewie, FCM, Shorebird) | Mobile = **React Native + Expo** (Zustand + TanStack Query, Expo Router, fetch + workspace api-client, Expo SQLite + Drizzle, react-native-webrtc, expo-video, Expo Notifications, EAS Update) | [ADR-0005](../adr/0005-mobile-react-native-vs-flutter.md) |
| Identity = Keycloak | Identity = **Clerk** (Keycloak deferred) | [ADR-0003](../adr/0003-identity-clerk-vs-keycloak.md) |
| Migration approach implied as full rewrite | Migration = **strangler-fig** from existing monolith | [ADR-0001](../adr/0001-strangler-fig-migration.md) |
| Repo layout informal | Repo layout = `apps/`, `services/`, `packages/`, `infra/`, `apis/`, `docs/` | [ADR-0002](../adr/0002-repository-layout.md) |
| Buyer + operator surfaces split named (ADR-009) | Same split, made concrete: `apps/web` (Next.js), `apps/admin` (Vite), `apps/partner` (Vite), `apps/studio` (Vite, new) | [ADR-0004](../adr/0004-web-framework-split.md) |
| Event backbone = Kafka (Redpanda) from day one | Event backbone = **Redpanda, gated**; DB outbox until second consumer appears | [ADR-0006](../adr/0006-event-backbone-redpanda.md) |
| Search = OpenSearch from day one | Search = **OpenSearch, gated** on catalog size; Postgres FTS until gate fires | [ADR-0007](../adr/0007-search-opensearch.md) |
| Analytics = ClickHouse + dbt from day one | Analytics = **ClickHouse + dbt, gated** on first analytics use case | [ADR-0008](../adr/0008-analytics-clickhouse-dbt.md) |
| Service mesh implied / unnamed | Service mesh = **Linkerd** | [ADR-0009](../adr/0009-service-mesh-linkerd.md) |
| Secrets via env vars | Secrets via **HashiCorp Vault**, phased migration | [ADR-0010](../adr/0010-secrets-vault.md) |

The remainder of v4.1 (target SLOs, NDPC posture, PCI SAQ-A scoping,
Lagos edge ingest tier, Hetzner FSN1/HEL1 topology, dual-payment-rail
strategy, Epplaa Boxes fulfillment, observability stack, sprint
mandate) is **retained verbatim**.

## 2. Phase plan

### Phase 0 — Decisions, ADRs, spec reconciliation (this PR)

- ADRs 0001–0010 written and accepted.
- This amendment file written.
- Supporting docs scaffolded: threat model, risk register, glossary,
  RACI, data dictionary, PCI CDF.
- Root files: `.nvmrc` (Node 24), `.github/CODEOWNERS`,
  `eslint.config.mjs` skeleton, `.well-known/security.txt`.
- Target directories created with `README.md` stubs.
- CI bumped Node 22 → Node 24.

### Phase 1 — Repository restructure

`git mv` the current directories into the v4.2 layout. CI must stay
green at every commit; runtime behaviour must not change.

| From | To |
| :--- | :--- |
| `artifacts/epplaa-app` | `apps/web-buyer-spa` |
| `artifacts/admin-console` | `apps/admin` |
| `artifacts/manufacturer-portal` | `apps/partner` |
| `artifacts/epplaa-mobile` | `apps/mobile` |
| `artifacts/api-server` | `services/api-monolith` |

Also create `apps/studio` (empty Vite + React scaffold) at this point
so it has a home before Phase 8 fills it.

### Phase 2 — Infrastructure-as-code foundation (2–3 sprints)

- `infra/terraform/` — Hetzner FSN1 (primary), HEL1 (DR), Cloudflare.
- `infra/terraform/modules/lagos-edge/` — Lagos ingest tier.
- `infra/k3s/` — k3s bootstrap (cloud-init, embedded etcd HA).
- `infra/helm/` — chart skeletons (ingress, Linkerd, Vault, Redpanda,
  OpenSearch, ClickHouse, Postgres operator, Redis, Argo CD,
  Prometheus + Grafana + Loki + Tempo, OTel Collector).
- `infra/argocd/applications/` — Argo CD `Application` manifests.
- `Dockerfile` for `services/api-monolith`; Helm chart deploys it as
  the **first cutover** to prove the substrate.
- Cloudflare Tunnel — only public ingress path.
- `deploy/runbooks/` — relocate; add cluster bootstrap, secret
  rotation, DR failover.
- CI builds container images, pushes to registry, renders Helm
  charts, runs `kubeval` and `helm lint`.

### Phase 3 — Cross-cutting platform capabilities (2 sprints; partly parallel to Phase 2)

- Vault deploy + first secret group migration (per ADR-0010 staging).
- OpenTelemetry actually wired (`@opentelemetry/sdk-node`, OTel
  Collector, Tempo for traces, Prometheus for metrics, Loki for
  logs, `/metrics` endpoint on the monolith).
- Grafana dashboards committed under `infra/grafana/dashboards/`.
- Linkerd installed; monolith namespace meshed; mTLS validated.
- Schema registry + Redpanda when the second-consumer trigger fires;
  `packages/events` introduced now with DB-backed implementation.
- OpenSearch when the catalog-size trigger fires; `packages/search`
  introduced now with Postgres-backed implementation.
- OPA bundle for cluster admission control + service-side authz
  starting with the existing MFA-elevation policy.

### Phase 4 — Service extraction (≈12 sprints, one per sprint)

Order:
1. notification-service
2. identity-service
3. catalog-service (introduces OpenSearch indexing if gate fires)
4. manufacturer-service
5. cart-service
6. payment-service (PCI scope reduction lives here)
7. order-service
8. fulfillment-service
9. discovery-service
10. stream-service
11. admin-service
12. analytics-service (introduces ClickHouse if gate fires)

Per-service template:

1. Create `services/<name>/` with `package.json`, `Dockerfile`,
   `src/`, `apis/openapi/<name>.yaml`, Helm chart, OTel + Vault
   integration, tests.
2. Move route handlers + libs + DB tables from monolith with `git mv`.
3. Establish service-owned schema (separate Postgres database or
   schema namespace).
4. Add gateway route at Traefik / Cloudflare so `/api/<domain>/*`
   targets the new service.
5. Run shadow traffic for one week.
6. Cut over; delete migrated code from `services/api-monolith`.

When the monolith is empty, delete it.

### Phase 5 — Live streaming architecture (3 sprints, parallel to Phase 4 once stream-service exists)

- `infra/streaming/lagos-edge/` — nginx-rtmp or SRS containers
  accepting RTMP + WHIP from sellers in Lagos.
- ffmpeg transcoding ladder (1080p / 720p / 480p / 240p) on edge
  nodes.
- Forward authoritative copy to **Cloudflare Stream** for global
  LL-HLS distribution + DVR.
- mediasoup SFU cluster in FSN1 for low-latency interactive paths
  (host ↔ co-host, moderator promotion).
- WHIP/WHEP signalling endpoints in `services/stream-service`.
- Replay pipeline: Cloudflare Stream → existing `replays` table →
  thumbnail extraction.
- RN mobile app and web buyer app updated to use WHIP for hosts and
  LL-HLS for viewers.

### Phase 6 — Buyer web migration to Next.js 15 (3 sprints)

- Scaffold `apps/web` (Next.js 15, App Router, React 19, Tailwind 4,
  shadcn/ui, next-intl, Zustand, TanStack Query).
- Port pages route-by-route. SPA stays at `spa.epplaa.com` until
  parity.
- Migrate auth bridge from Clerk SPA SDK to Clerk Next.js SDK.
- SSR for product / stream / seller pages (SEO + share previews).
- `next-intl` with English plus Yoruba, Igbo, Hausa scaffolding.
- Lighthouse + Core Web Vitals budgets in CI.
- At parity, redirect `spa.epplaa.com` to `apps/web` and delete
  `apps/web-buyer-spa`.

### Phase 7 — Mobile (RN/Expo) buildout (4 sprints, parallel)

Buildout uses the package mapping in
[ADR-0005](../adr/0005-mobile-react-native-vs-flutter.md). Targets
the spec §5.5 budgets verbatim:

- Cold start ≤ 2.0 s on Pixel 4a / equivalent mid-tier Android.
- Sustained 60 fps video + UI thread on stream watch screen.
- Install size ≤ 30 MB Android APK / 50 MB iOS IPA.
- Memory ceiling ≤ 220 MB during stream watch.

Scope: Sentry RN SDK; Paystack RN + Flutterwave RN; Expo
Notifications (FCM + APNs); permissions flow; deep linking; offline
cache via Expo SQLite + Drizzle; WHIP host with react-native-webrtc;
LL-HLS viewer with expo-video; EAS Update OTA.

### Phase 8 — Operator / studio / partner apps (2 sprints)

- `apps/admin` migrated to TanStack Router; otherwise unchanged.
- `apps/studio` populated by extracting seller/studio + go-live +
  replays + earnings pages out of `apps/web-buyer-spa`.
- `apps/partner` migrated to TanStack Router; otherwise unchanged.

### Phase 9 — Quality engineering (continuous; formalised over 2 sprints)

- Pact contracts replace hand-rolled `mfa.contract.test.ts`; one
  consumer-provider pair per service boundary as services land.
- Playwright E2E: wire root `e2e.test.ts` into CI; expand to top
  buyer flows (browse → cart → checkout → payment → order,
  host-go-live, admin-takedown).
- Maestro: `.maestro/` flows for the RN app (sign-in, browse, watch,
  buy).
- k6 load tests at `tools/load/`; baseline RPS for catalog read,
  payment webhook, stream chat.
- Coverage gate raised from `passWithNoTests` to per-package
  thresholds (services 80% lines, packages 90%).
- CodeQL added to CI alongside Semgrep.
- Trivy switched from filesystem scan to per-image container scan
  once images exist.

### Phase 10 — Compliance, beta, launch (3 sprints)

- Complete `docs/compliance/pci-cdf.md` — confirm SAQ-A scope (no PAN
  ever touches our infra).
- Complete `docs/threat-model.md` — STRIDE per service.
- Auto-generate `docs/data-dictionary/` from Drizzle schemas.
- Retention schedule + automated deletion jobs.
- DR drill: failover from FSN1 to HEL1; document RPO/RTO actuals.
- Incident response rehearsal using existing `rehearse-*` workflows.
- Beta cohort + feature flags.
- Launch checklist signoff (matches v4.1 spec §11).

## 3. Explicit non-goals

- **Flutter migration** — superseded by ADR-0005.
- **Keycloak migration during this program** — superseded by
  ADR-0003.
- **Replatforming away from Replit dev environment** — Replit
  remains the dev sandbox; production runs on Hetzner k3s.

## 4. Acceptance

This amendment is accepted when the same approval table as v4.1
§4 (CTO, Head of Security, Head of SRE, Head of Product, DPO,
Finance/Compliance) signs off. Until that signoff, ADRs 0001–0010
hold individually but the *program* is provisional.
