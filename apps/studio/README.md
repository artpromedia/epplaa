# `apps/studio` — Seller Studio (placeholder)

Per ADR-0004 (Web framework split) and Phase 8 of the v4.2 amendment,
`apps/studio` will be a **dedicated Vite + React SPA** containing the
seller / studio surface (go-live composer, replays, earnings,
moderation tools).

## Status

**Placeholder.** No source has landed yet. The seller/studio pages
currently live inside `apps/web-buyer-spa/` and will be carved out
into this app in Phase 8 of the v4.2 plan
(`docs/architecture/v4.2-amendment.md`).

This directory exists so that:

- `pnpm-workspace.yaml` already resolves the package name on workspace install.
- CODEOWNERS, ADRs, and the v4.2 amendment can reference a stable path.
- Phase 8 lands as a *fill-in*, not a *new directory*, which keeps
  diffs reviewable.

## What lives here when populated

- `package.json` — `@workspace/studio`, Vite + React + TanStack Router + TanStack Query.
- `src/` — go-live composer, replay manager, earnings dashboard, T&S workflows.
- `vite.config.ts`, `vitest.config.ts`, `tsconfig.json` mirroring `apps/admin/` conventions.
- `.replit-artifact/artifact.toml` for Replit deploy parity.

Until then, do **not** add scaffolding here — the eventual scaffold is
generated by `pnpm create vite` against the same React/Tailwind/shadcn
stack used by `apps/admin/`, and pinning a hand-rolled scaffold now
would only be code we throw away when Phase 8 starts.
